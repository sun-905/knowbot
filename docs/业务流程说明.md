## 用户注册与登录流程

```
注册: POST /auth/register
  ├─ 校验: phone必填, password≥6位, nickname必填
  ├─ 查重: phone/email 唯一
  ├─ 密码: bcrypt 哈希存储
  └─ 返回: JWT token (24h有效)

登录: POST /auth/login
  ├─ 输入: 手机号或邮箱 + 密码
  ├─ 查询: (phone == account) OR (email == account)
  ├─ 校验: bcrypt 验证密码, 检查 is_active
  └─ 返回: JWT token + 用户信息
```

## 知识库文档管理流程

```
管理员上传文档
  │
  ▼
POST /knowledge/docs/upload (multipart)
  ├─ 校验: 文件类型 (PDF/TXT/MD), 大小 (≤10MB)
  ├─ create_document: 落盘 tmp_uploads/ + DB记录 (status=processing) → 201 立即返回
  └─ BackgroundTasks.process_document:
       ├─ 解析 (PDF 三级回退):
       │   L1: SimpleDirectoryReader (文本型PDF)
       │   L2: PyMuPDF 原生文本提取 (含文字层的复杂PDF)
       │   L3: PaddleOCR (扫描件/图片型PDF，含 PP-Structure 表格识别)
       │   非 PDF: SimpleDirectoryReader 直接解析
       ├─ SentenceSplitter 切片 (512字, 50重叠)
       ├─ BGE 向量化 (batch)
       ├─ Qdrant upsert (batch_size=32)
       ├─ BM25 增量更新
       └─ status → ready / failed (+ error_msg)

系统启动时:
  └─ process_pending_documents: 扫描 status=processing
       ├─ 有临时文件 → 继续处理
       └─ 临时文件丢失 → 标记 status=failed

BM25 索引:
  └─ rebuild_bm25_from_qdrant: 每次启动从 Qdrant 全量重建
```

## 反馈流程

```
用户对 AI 回答评价
  │
  ├─ 👍 点赞 → POST /messages/{message_id}/feedback { rating: "like" }
  │              ├─ 同一消息仅可评价一次 → 409
  │              └─ 成功 → 201
  │
  └─ 👎 踩   → 展开文字输入框（选填, ≤500字）
                 └─ POST /messages/{message_id}/feedback { rating: "dislike", comment? }
                    └─ 管理员可在后台查看
```

## LLM 容错流程

```
LLM 调用
  ├─ 尝试 DeepSeek (主)
  │   ├─ 成功 → 返回结果
  │   └─ 失败 → 1s 后重试
  │       ├─ 成功 → 返回结果
  │       └─ 失败 → 2s 后重试
  │           ├─ 成功 → 返回结果
  │           └─ 失败 → 切换 Qwen (备)
  │
  ├─ 尝试 Qwen (备)
  │   ├─ 成功 → 返回结果
  │   └─ 失败 → 返回静态兜底:
  │       "抱歉，我暂时无法回答这个问题。建议您联系人工客服获取帮助。"
  │
  └─ [熔断器] 连续5次失败 → 冷却60s，期间直接返回兜底回复
```

## 配额与并发控制

```
用户发消息
  ├─ 每日配额: daily_usage 表 user_id+usage_date 唯一约束
  │   ├─ 今日首条 → INSERT count=1
  │   ├─ 已有记录 → UPDATE count+1 (需 count < daily_quota)
  │   └─ 已用完 → 429 "今日提问次数已用完"
  │
  └─ 并发限制: 内存计数器 (≤3 连接/用户)
      ├─ 有空位 → 计数+1，允许进入
      ├─ 满 → 429 "请等待上一轮对话完成"
      └─ SSE 结束 → asyncio.shield 保护清理，计数-1
```
