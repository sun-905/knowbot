# AI 智能客服系统 — 运行指南

> 最后更新：2026-06-26 | 适用版本：v1.0

---

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Docker Desktop | 最新稳定版 | 运行 MySQL 8.0 + Qdrant 向量数据库 |
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建与开发 |
| npm | 9+ | 前端包管理器 |
| Git | 任意版本 | 版本控制（可选） |

> **Windows 用户**：建议使用 Git Bash 或 WSL2 执行命令。所有路径使用正斜杠 `/`。

---

## 2. 获取 API Key

### 2.1 DeepSeek API（必需）

1. 访问 [platform.deepseek.com](https://platform.deepseek.com) 注册/登录
2. 进入「API Keys」页面，点击「创建 API Key」
3. 复制生成的 Key（格式：`sk-xxxxxxxx`）
4. **首次注册通常赠送免费额度，无需充值即可开发测试**

### 2.2 Qwen API（可选，用于降级备份）

1. 访问 [dashscope.aliyun.com](https://dashscope.aliyun.com) 注册
2. 获取 API Key，填入 `.env` 中 `QWEN_API_KEY`

---

## 3. 配置环境变量

```bash
# 在项目根目录
cp .env.example .env
```

编辑 `.env`，将 `DEEPSEEK_API_KEY` 替换为你的真实 Key：

```bash
DEEPSEEK_API_KEY=sk-你的真实Key
```

> ⚠️ `.env` 已在 `.gitignore` 中，不会被提交到 Git。切勿将真实 Key 写入 `.env.example`。

---

## 4. 启动基础设施

```bash
# 在项目根目录
docker compose up -d
```

首次运行会自动拉取镜像（MySQL ~500MB，Qdrant ~200MB），耗时取决于网络。

验证服务状态：

```bash
docker compose ps
```

期望输出：`cs_bot_mysql` 和 `cs_bot_qdrant` 均为 `healthy` / `running`。

> MySQL 首次启动会自动执行 `backend/sql/init.sql` 建表。健康检查含密码认证，确保 `mysqladmin ping` 返回 OK。

---

## 5. 启动后端

```bash
cd backend

# 5.1 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate      # Linux/Mac
# venv\Scripts\activate       # Windows CMD

# 5.2 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 5.3 启动开发服务器
uvicorn app.main:app --reload --port 8000
```

**首次启动注意事项**：
- BGE Embedding 模型（`bge-large-zh-v1.5`）首次自动下载，约 1.3GB，请耐心等待
- BM25 索引从 Qdrant 全量重建（首次为空，上传文档后生效）
- 控制台会打印检索阈值校准警告——这是预期行为，生产环境前需校准

**验证**：浏览器打开 http://localhost:8000/docs 查看自动生成的 API 文档。

---

## 6. 启动前端

```bash
cd frontend

# 6.1 安装依赖
npm install

# 6.2 启动开发服务器
npm run dev
```

**验证**：浏览器打开 http://localhost:5173 ，应看到登录页面。

> 开发服务器默认连接 `localhost:8000` 后端。若后端地址不同，修改 `src/api/client.ts` 中的 `baseURL`。

## 7. 端到端验证

### 7.1 注册账号

1. 打开 http://localhost:5173/register
2. 填写手机号或邮箱 + 密码 → 注册
3. 注册成功后自动跳转到对话页

### 7.2 上传知识库文档

1. 点击导航栏「知识库」（仅管理员可见；首个注册用户需手动在 MySQL 中设置 `is_admin=1`）
2. 上传一个测试文档（PDF/TXT/Markdown 均可）
3. 等待状态变为「就绪」

**设置管理员**：
```sql
-- 通过 MySQL 客户端执行
UPDATE users SET is_admin = 1 WHERE id = 1;
```

### 7.3 测试 RAG 问答

1. 进入对话页，输入文档相关的问题
2. 观察：意图标签 → 参考来源 → 流式逐字回答 → 追问建议
3. 测试检索失败场景：问一个文档中没有的内容 → 应返回兜底话术

### 7.4 测试管理后台

1. 管理员账号登录 → 点击「管理」
2. 查看日均问答量 / 意图分布 / 反馈统计

## 8. 常见问题排查

### Q1: Docker 容器启动失败

```bash
# 查看日志
docker compose logs mysql
docker compose logs qdrant

# 端口冲突：修改 docker-compose.yml 中的端口映射
# 3306 冲突 → 改为 "3307:3306"
# 6333 冲突 → 改为 "6335:6333"
```

### Q2: 后端报 "ModuleNotFoundError: No module named 'xxx'"

```bash
# 确认在 backend 目录且有虚拟环境
cd backend
pip install -r requirements.txt
```

### Q3: 前端白屏 / API 请求失败

- 确认后端在 8000 端口运行：`curl http://localhost:8000/health`
- 检查浏览器控制台是否有 CORS 错误
- 确认 `.env` 中 DeepSeek API Key 有效且有余额

### Q4: BGE 模型下载慢/失败

```bash
# 方案1：使用镜像
export HF_ENDPOINT=https://hf-mirror.com

# 方案2：手动下载后指定本地路径
# 下载 https://huggingface.co/BAAI/bge-large-zh-v1.5 到本地目录
# 修改 .env：EMBEDDING_MODEL=/path/to/local/model
```

### Q5: DeepSeek API 返回 401/429

- 401：API Key 无效或过期，重新生成
- 429：请求频率超限或余额不足，稍后重试或充值
- 系统会自动重试 3 次（指数退避），连续失败 5 次后进入 60 秒冷却期

### Q6: BM25 检索返回空结果

- BM25 索引在启动时从 Qdrant 重建，确保 Qdrant 已启动且有数据
- 重启后端会自动重建 BM25 索引
- 检查启动日志中是否有 "BM25 index rebuilt" 信息

## 9. 停止与清理

```bash
# 停止所有服务
docker compose down

# 停止并删除数据卷（⚠️ 所有数据将丢失）
docker compose down -v

# 删除后端虚拟环境
rm -rf backend/venv

# 删除前端依赖
rm -rf frontend/node_modules
```

## 10. 开发模式 vs 生产模式

| 配置项 | 开发模式 | 生产模式建议 |
|--------|---------|-------------|
| `JWT_SECRET` | 默认值 | 随机生成 32 位字符串 |
| `EMBEDDING_DEVICE` | `cuda`（自动降级 CPU） | `cuda`（确保有 GPU） |
| `RETRIEVAL_THRESHOLD` | `0.43` | 经评测集校准后的值 |
| CORS origins | `localhost:5173` | 实际域名 |
| MySQL 密码 | `root123` | 强密码 + 环境变量注入 |
| `.env` | 本地文件 | Secret Manager / K8s Secret |
